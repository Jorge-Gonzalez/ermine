PART A — ANALYSIS PASS: how much of the "identity" residue is actually non-Ermine?
========================================================================

Method — the two-metric invariance test (docs/proportional-plane.md). For each residue
declaration, ask: would the value survive if the design were re-rendered at a different
resolution?
  R  relational  — survives (a proportion, ratio, order, edge, alignment) → a WORD
  S  scale-backed — rides the grain (var(--spacing-*), var(--radius-*)) → EXPRESSIBLE via
                    existing/near spacing/corner grammar
  O  off-grid    — neither (a raw px/rem/em not on any scale) → GENUINE identity (the floor)
  X  substrate/reset — box-sizing, margin:0, auto, mechanism → not a positive carrier

PART C — RULE-ACTION CLASSIFICATION (what the declarations are doing)
========================================================================

Class names are intentionally ignored here. The new structure should come from the
declarations: what they change, which visual/mechanical function they serve, and whether
that function wants an Ermine word, a recipe molecule, or a local component drawing.

1. Dimension and constraint rules.
Function: set a component's physical envelope: fixed inline size, square control size,
minimum interaction height, viewport-bounded dialog size, or scrollable result height.

Representative declarations:
  width: 320px; width: 280px; width: min(600px, calc(100vw - 2rem));
  height: min(560px, 85vh); height: 16px / 18px / 28px; width: 24px / 2rem / 3rem;
  min-width: 36px / 200px / 240px / 26px; max-width: 8em / 360px / none;
  min-height: 48px / 150px / 0; max-height: 256px / 400px;
  flex: 0 0 auto; flex: 1.

Reading:
  - The relational pieces are already Ermine-shaped: fill, viewport bounds, rigid/grow,
    and min-height escapes.
  - The repeated raw constants are the real size vocabulary gap: icon/control squares,
    popover widths, editor command width, dialog bounds, kbd minimums, and result caps.
  - These should become size/measure words only where the same physical intent repeats
    across components. One-off dialog or popover measurements can remain recipe identity.

Likely Ermine pressure:
  measured width/height scale; square-size/icon-size words; dialog/popover measure recipe;
  scroll-cap/max-block words; control-min-block.

2. Attachment, edge, and layer rules.
Function: attach a child to an edge, place a floating affordance, or assert stack order.

Representative declarations:
  left: 0; right: 0; top: 100%; top: calc(100% + 4px);
  bottom: 52px; right: var(--spacing-sm); top: 0; bottom: 0;
  z-index: 0 / -1 / 10000 / 2147483646; position: fixed !important;
  inset: 0 2px 4px; pointer-events: none.

Reading:
  - `left:0/right:0/top:100%` is dropdown attachment, not component identity.
  - arrow top/bottom placement and edit-button right offset are edge-attachment variants.
  - Ordinary in-page dropdown/content z-index values are now z-scale words. Remaining
    values are extension/top-layer ordering or pseudo negative drawing.
  - `bottom:52px` is a local toast avoidance offset and probably recipe-local unless a
    general "float above bar" pattern repeats.

Likely Ermine pressure:
  edge-attach words; below/above attachment; layer tiers; inset member/facet; float-above
  recipe for transient bars.

3. Spacing and rhythm rules.
Function: distribute internal padding and external rhythm, especially where current
spacing words still need a reset, shorthand-asymmetry, or raw micro-geometry ruling.

Representative declarations:
  padding: var(--spacing-2xl) 0 var(--spacing-xl);
  padding: var(--spacing-md) var(--spacing-sm) 0 0;
  padding: 3px 6px 6px; padding: 1px 4px; padding: 0;
  margin: 0; margin-top: var(--spacing-sm); margin-right: 1rem / var(--spacing-xl);
  margin-left: var(--spacing-sm);
  gap: 2px.

Reading:
  - Symmetric two-axis padding/margin now uses existing Ermine inline/block words. The
    remaining scale-backed padding/margin wants finer facets: start/end and before/after,
    not more component classes.
  - The raw 3px/6px/1px/4px keycap and suggestion-chip padding is micro-object drawing;
    it is probably a recipe unless the design wants a tiny density scale.
  - `gap:2px` repeats inside menus/dropdowns as a very small cluster gap. This could be a
    `gap-2xs` style scale member if admitted.

Likely Ermine pressure:
  per-edge spacing words; zero-padding/reset words; smaller gap/density step; keycap/chip
  recipe for raw micro-padding.

4. Surface, line, elevation, and cutout rules.
Function: paint or remove surfaces, borders, shadows, separators, and the transparent
sentinels that make state changes possible.

Representative declarations:
  background-color/background: var(--shadow-color), var(--tone), var(--tone-dim),
    var(--accent), transparent, color-mix(...);
  color: var(--accent), var(--ink-alt), var(--ink-soft), inherit;
  border: none; border-top: none; border-bottom: 2px solid transparent;
  border-bottom-color: var(--accent); border-right: none;
  border-right-color: transparent; border: 1px solid var(--tone-dim);
  border-radius: var(--radius-md), var(--radius-sm), 4px, 2px, 0 0 md md;
  border-bottom-left/right-radius: 0 / var(--radius-md);
  box-shadow: modal/elevated shadows; mix-blend-mode: multiply.

Reading:
  - Many color/surface rows are already conceptually Ermine skin words, but remain local
    because selector specificity or pseudo drawing requires a local hook.
  - Transparent border sentinels and "remove one side of the rule" are rule mechanics.
    They want line-presence facets, not named component classes.
  - The modal shadow/multiply blend and suggestion arrow border trick are recipe identity.
  - The validation focus inset shadow is state skin, likely a focus/error treatment word.

Likely Ermine pressure:
  side-specific rule presence; transparent rule sentinel; state skin facets; elevation

Representative declarations:

Reading:

  - nowrap/pre-wrap are now ruled text-flow words; list reset/list restore remain reusable

  - The mark/highlight treatment (`mark span { opacity: .35 }`) is search-result recipe.

Likely Ermine pressure:

6. Interaction affordance and state rules.
Function: signal hover, selected, active, disabled, current, checked, focus/error, and
minimum-selection states.

Representative declarations:
  cursor: pointer / not-allowed; user-select: none;
  background-color on hover/selected/checked/disabled/current;
  color/border-color on selected/current/checked;
  opacity: 0.6 / 0.9 / 0.95 / 1;
  transform: scale(0.98); vertical-align: middle;
  .is-selected, [aria-current], [aria-checked], [aria-selected], :disabled, :hover,
    :active, :focus, :only-of-type.

Reading:
  - The state grammar is the biggest signal here: many local rules are not component
    identity, they are "when selected/current/disabled, apply skin/affordance."
  - `only-of-type` for minimum selection is business behavior made visible; it can remain
    a small recipe or become a min-selection state word if reused.
  - vertical icon alignment inside segmented options is component-local unless a general
    "inline-icon-middle" word is admitted.

Likely Ermine pressure:
  condition-backed state facets; disabled affordance; selected/current/checked skin;
  min-selection recipe; press/active affordance.

7. Motion and transition rules.
Function: define which properties animate, at what tempo, and when animation is suppressed.

Representative declarations:
  transition: left 0.1s ease, width 0.1s ease;
  transition: none !important.

Reading:
  - `tween-quick` handled universal transition rows, and R-MOTION-09 targeted tween
    envelopes handled the repeated border/ground/ink/opacity/transform target sets.
  - segmented-control pill movement is a bespoke mechanism because it animates CSS custom
    coordinate variables and snaps before sliding.
  - `transition:none !important` on `shake` is an effect-composition override: suppress
    local tweens so the generated shake effect can own the frame.

Likely Ermine pressure:
  tween-none/suppress-tween for effect composition; recipe-local segmented pill timing.

8. Reset, inheritance, and neutralization rules.
Function: remove browser/component defaults or force inheritance so Ermine words can take
over without fighting UA or previous recipe styling.

Representative declarations:
  box-sizing: border-box; font-family: inherit; border: none; background: transparent;
  margin: 0; padding: 0; list-style: none; width: auto; max-width: none;
  border-top: none; border-right: none; border-radius: 0;
  min-height: 0; overflow: visible; text-overflow: clip.

Reading:
  - These are usually not new positive words. They are reset/substrate escape hatches or
    side-effect removals.
  - Repeated neutralization might deserve explicit negative words only where authors need
    to opt out in markup: unbordered, transparent, unrounded-side, unbounded, untruncate.

Likely Ermine pressure:
  reset layer coverage; negative/escape facets for border/background/radius/truncation;
  min-content escape already covered by `min-height-none`.

9. Component-private drawing recipes.
Function: draw a specific object whose geometry is not a general layout word.

Representative declarations:
  triangle arrow: width:0; height:0; border:6px solid transparent; border-top/bottom-color.
  keycap bevel: ::after content; inset: 0 2px 4px; border; background; z-index:-1.
  segmented pill: ::before content; absolute fill; custom left/width variables.
  modal blend/shadow pair; search mark fade; placeholder pseudo-element.

Reading:
  - These are the strongest "keep local or recipe" candidates. They are not class names
    looking for assimilation; they are small drawings made from CSS primitives.
  - If reused broadly, they should enter Ermine as recipes/molecules, not as flat utility
    words.

Likely Ermine pressure:
  kbd/keycap recipe; arrow/callout recipe; segmented-control recipe; placeholder recipe.

Summary shape for the next assimilation pass:
  - Most likely words: edge attachment, layer tiers, side-specific rule/radius resets,
    measured size scale.
  - Most likely molecules/recipes: prose/editor content, keycap, callout arrow, segmented
    control pill, modal/dialog measure, search highlight.
  - Most likely to stay local: one-off offsets, modal blend/shadow identity, raw micro
    drawing constants, business-specific min-selection behavior.

PART D — ASSIMILATION PLAN FROM RULE ACTIONS
========================================================================

Reframe the headline first: "0 assimilable" means "0 declarations match words that
already exist," not "0 declarations are generalizable." The rule-action pass shows a
large latent grammar layer: declarations that express broad layout, scale, state, motion,
or recipe behavior but have not yet been admitted into Ermine.

Planning principle: do not reject a declaration family merely because its current Monky
value is raw or component-named. First ask whether the function can be expressed through
an existing Ermine scale, a proportional relation, a missing facet of an existing word, or
a molecule/recipe with sockets. Dimensions are the clearest example: `320px`, `28px`, and
`min(600px, calc(100vw - 2rem))` are not automatically local identity. They may be scale
members, measure roles, bounded-container relationships, icon/control sizes, or recipe
sockets.

1. Reclassify the residue.

Replace the flat "Monky-owned" reading with review buckets that describe why a rule is
not assimilated yet:
  - admitted: already expressible by current Ermine words.
  - latent-word: general behavior, but no Ermine word exists yet.
  - latent-facet: the concept exists, but an axis is missing (edge, side, property, state).
  - latent-scale: a raw value that can map to an existing or proportional Ermine scale.
  - recipe: reusable molecule, not a flat utility word.
  - local-identity: genuinely Monky-specific after the invariance and scale tests.

This makes future reports say:
  assimilable-now: N
  latent-generalizable: N
  likely-recipe: N
  likely-local: N

3. Pilot the dimension and constraint family first.

Do not treat size constants as identity by default. Sort the rules by function:
  - control/icon squares: 16px, 18px, 24px, 28px, 2rem, 3rem.
  - measure widths: 200px, 240px, 280px, 320px, 360px.
  - bounded containers: min(600px, calc(100vw - 2rem)), min(560px, 85vh).
  - scroll caps: 256px, 400px.
  - interaction floors: 48px, 150px.
  - proportional constraints: 85vh, 100vw - 2rem, 8em.

Then test each against Ermine's existing scales and proportional relationships before
declaring it local. Possible outputs:
  square-xs/sm/md/lg; icon-sm/md/lg; measure-sm/md/lg; max-block-sm/md;
  dialog-measure; popover-measure; control-min-block; editor-min-block.

4. Admit missing facets before inventing large concepts.

Several rule families look like current Ermine concepts with one missing axis:
  - line/reset facets: transparent rule sentinels and negative release rows that cannot be
    replaced by positive rule-edge facts.
  - side-specific radius: corner-bottom-md, corner-top-md, corner-bottom-none.
  - edge attachment: attach-below, attach-above, stretch-inline, edge-end-sm.
  - layer tiers: layer-modal, layer-popover, layer-local.
  - targeted tweens: done for repeated Monky target sets; only suppression/stagger
    questions remain.
  - state skin: selected/current/checked/disabled/focus-error facets.

These should be evaluated before adding broad new planes.

5. Promote repeated drawings as recipes, not flat utilities.

The following are broad enough to preserve, but too shape-specific for atomic words:
  - prose/editor-content molecule.
  - keycap/kbd recipe.
  - callout arrow recipe.
  - segmented-control pill recipe.
  - modal/dialog measure and frame recipe.
  - search-highlight recipe.

Recipes should expose sockets for scale, skin, state, and motion where the pattern varies.

6. Process the remaining categories in priority order.

Recommended queue:
  1. selected/current/checked/disabled state skin.

The order is chosen to reduce residue while strengthening Ermine's grammar, not merely to
