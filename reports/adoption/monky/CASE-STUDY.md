# Monky / Ermine adoption case study

Monky was Ermine's first full application adoption: a real extension with injected overlays,
page styles, Shadow DOM delivery, generated grammar CSS, theme sockets, and local component
recipes. The point was not to maximize conversion percentage. The point was to conserve every
declaration, move only what Ermine could lawfully own, and turn everything else into an explicit
boundary or a filed question.

## What Monky Yielded Ermine

The test bench produced concrete Ermine rulings and vocabulary:

- Conditioned skin moved from hover-only evidence through backed conditions: `selected:`,
  `checked:`, `focus:`, and `current:`.
- `elevated` was admitted with an explicit boundary for identity shadows and focus rings.
- R-SKIN-10 named the recipe boundary: project bundles such as buttons, inputs, cards, alerts,
  sections, segmented controls, and selectable groups consume sockets but remain product identity.
- The theme socket plane was validated against a real palette resolver and a generated CSS
  consumer.
- Gap reports now hold counted evidence for animation timing, scrollbar prominence,
  parent-relational state, interaction affordance, and skin-surface mechanics (the dataviz
  palette question was filed earlier, without Monky rows).

## What Stayed Local

The current boundary is generated at `reports/adoption/monky/BOUNDARY.md`. It declares the live
state machine-checked by `adoption/current-ledger.ts --gate`:

| measure | result |
|---|---:|
| current declarations | 714 |
| project-owned residue | 525 |
| assimilable declarations | 0 |
| review-coded declarations | 0 |

The residue is not leftover adoption work. It is recipe identity, exact geometry, brand type,
component mechanics, user-content defaults, reset/absence mechanics, or pre-counted gap evidence.

## Accounting Method

The frozen ledger and current ledger answer different questions.

The frozen ledger (`ledger.json`) conserved the starting U2 baseline. It was necessary because
deleting or rewriting declarations during pilots would otherwise erase evidence. That is the
"stuck 339" lesson: the old `skin-local` number was historically true, but it stopped describing
the live code after Monky adopted generated grammar and socketed skin words.

The generated current ledger (`current-ledger.json` and `CURRENT-LEDGER.md`) answers the live
question: what does the current CSS still contain, and can emitted Ermine vocabulary express any of
it now? It is selector-aware, condition-aware, bridge-aware, and reason-coded. Phase D made the last
step explicit by failing the gate when `assimilable > 0` or any review bucket is non-empty.

## Lessons

Literal matching drifts. The reliable comparison was emitted Ermine CSS parsed back into
declarations, with values resolved through Monky's own scale bindings and bridge aliases.

Cascade layers matter. Phase C exposed that a local component-layer declaration can outrank
generated grammar even when both use the same socket, so a token match is not automatically an
effective rendering change.

Names need evidence. The raised/elevated collision showed that a good treatment can still have a
bad word until the surrounding vocabulary is checked.

Evidence-gated admission worked. Monky could ask Ermine hard questions without smuggling local
decisions into shared grammar. When evidence was strong, Ermine changed; when it was not, the row
became a boundary or a Gap Report.
